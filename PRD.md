# PRD: 의사랑 EMR 신규 입사자 온보딩 프로그램

## 1. 개요

### 1.1 프로젝트 명
의사랑 EMR 온보딩 프로그램

### 1.2 목적
신규 입사자가 의사랑 EMR 시스템을 빠르고 효과적으로 학습할 수 있도록 인터랙티브 튜토리얼 기반의 온보딩 프로그램을 제공한다.

### 1.3 대상 사용자
- 병원/의원 신규 입사 직원
- EMR 시스템 신규 사용자
- 진료실, 접수실, 보험청구 담당자

### 1.4 핵심 가치
- **실습 중심 학습**: 실제 화면과 동일한 인터랙티브 시뮬레이터
- **단계별 가이드**: 사용자가 직접 클릭하며 배우는 튜토리얼
- **즉시 피드백**: 각 단계 완료 시 시각적 피드백 제공

## 2. 기능 요구사항

### 2.1 탭 구성

| 탭 | 설명 |
|---|---|
| 🏠 홈 | 대시보드, 학습 진행률, 튜토리얼 목록 |
| 📚 튜토리얼 | 인터랙티브 시뮬레이터 기반 학습 |
| 📝 퀴즈 | 학습 내용 확인 테스트 |
| 🔍 질문검색 | FAQ 검색 및 카테고리별 필터 |

### 2.2 튜토리얼 목록

#### 진료실
| ID | 제목 | 설명 | 단계 수 |
|----|------|------|---------|
| rx1 | 환자진료기록생성 | 환자 선택 → 상병/처방 입력 → 계산 → 저장 | 6단계 |
| rx2 | AH200 교육기록 | 만성질환관리료 산정 후 교육기록 작성 | 4단계 |
| rx3 | 비대면 비율 | 외래완료에서 비대면 비율 확인 | 3단계 |

| ID | 제목 | 설명 | 단계 수 |
|----|------|------|---------|
| rec1 | 외래 접수 | 신규 환자 등록 → 접수 → 진료대기 확인 | 7단계 |
| rec2 | 수납 처리 | 본인부담금 수납 및 영수증 출력 | 4단계 |

#### 보험청구
| ID | 제목 | 설명 | 단계 수 |
|----|------|------|---------|

### 2.3 인터랙티브 기능

#### 시각적 가이드
- **빨간색 하이라이트**: 클릭해야 할 요소 강조
- **단계 배지**: 현재 단계 번호 표시 (빨간 원형)
- **말풍선**: 수행할 액션 안내
- **빨간 박스**: 중요 확인 영역 강조

#### 진행 표시
- 좌측 사이드바: 전체 학습 목차
- 시나리오 패널: 현재 튜토리얼 단계 목록
- 완료된 단계: 초록색 표시
- 현재 단계: 파란색 강조

## 3-A. 비기능 요구사항

## 3-B. 사용자 스토리 및 인수 기준

### US-001: 튜토리얼 단계 진행
**As** 신규 직원
**I want** 클릭만으로 EMR 화면을 단계별로 따라가고 싶다
**So that** 강사 없이도 업무 흐름을 익힐 수 있다

**인수 기준:**
- [ ] 클릭 대상 요소가 빨간 하이라이트 + 단계 배지로 표시된다
- [ ] 잘못된 영역 클릭 시 현재 단계가 유지된다
- [ ] 튜토리얼 완료 시 초록색 축하 오버레이가 표시된다
- [ ] 완료된 단계는 좌측 패널에 초록 체크로 표시된다

### US-002: 진행률 영속성
**As** 신규 직원
**I want** 브라우저를 닫고 다시 열어도 진행 상황이 유지되길 바란다
**So that** 여러 날에 걸쳐 학습을 이어갈 수 있다

**인수 기준:**
- [ ] 튜토리얼 완료 후 홈 화면 진행률이 즉시 업데이트된다
- [ ] 페이지 새로고침 후에도 완료된 튜토리얼 수가 유지된다
- [ ] 사이드바에 완료된 튜토리얼 ✓ 체크가 유지된다

### US-003: 퀴즈 학습 확인
**As** 신규 직원
**I want** 튜토리얼에서 배운 내용을 퀴즈로 확인하고 싶다
**So that** 실제 업무 전 이해도를 점검할 수 있다

**인수 기준:**
- [ ] 8문제가 순서대로 출제된다
- [ ] 답 선택 즉시 정오 여부가 색상으로 표시된다
- [ ] 최종 점수가 퍼센트로 표시된다
- [ ] "다시 풀기" 버튼으로 재시도 가능하다

## 3-C. ROI 분석

### 기존 방식 vs 이 프로그램

| 항목 | 기존 집합 교육 | 이 프로그램 |
|------|-------------|-----------|
| 교육 시간 | 4~8시간 (강사 포함) | 30분 이내 자습 |
| 재수강 가능성 | 어려움 (일정 조율) | 언제든 반복 가능 |
| 강사 비용 | 시간당 비용 발생 | 없음 |
| 오조작 위험 | 실제 환경 실습 → 위험 | 시뮬레이터 → 안전 |
| 교육 일관성 | 강사별 편차 | 항상 동일한 내용 |

### 예상 효과
- **교육 시간 75% 단축** (8시간 → 2시간)
- **오조작 사고 감소** (시뮬레이터 반복 학습)
- **온보딩 만족도 향상** (자기 주도 학습)

## 4. 기술 스택

- **HTML5**: 단일 파일 구조
