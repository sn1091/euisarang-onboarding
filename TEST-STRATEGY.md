## 2. 테스트 피라미드

```
         /\
        /E2E\         ← 사용자 시나리오 전체 플로우 (Playwright)
       /------\
      /통합테스트\     ← 모듈 간 연동 (Jest + jsdom)
     /----------\
    /  단위테스트  \   ← 순수 함수 / 데이터 검증 (Jest)
   /--------------\
```

| 계층 | 비중 | 도구 | 목적 |
|------|------|------|------|
| 단위 | 60% | Jest | 퀴즈 채점 로직, FAQ 필터, 시나리오 데이터 구조 |
| 통합 | 25% | Jest + jsdom | 탭 전환, 진행률 추적, 튜토리얼 단계 흐름 |
| E2E  | 15% | Playwright | 실제 브라우저에서 전체 사용자 여정 |

## 3. 테스트 파일 구조

```
tests/
├── helpers/
│   └── data.js                     # 공유 데이터 (QUIZ_DATA, FAQ_DATA, SCENARIOS)
├── unit/
│   ├── quiz.test.js                # 퀴즈 채점·점수·메시지 로직
│   ├── faq.test.js                 # FAQ 검색·카테고리 필터 로직
│   └── scenario-data.test.js      # 시나리오 데이터 무결성
├── integration/
│   ├── tutorial-flow.test.js      # 튜토리얼 단계 진행 및 진행률 추적
│   └── navigation.test.js         # 탭 전환 및 시나리오 네비게이션
└── e2e/
    └── full-onboarding.spec.js    # 전체 사용자 여정 E2E
```

## 4. 단위 테스트 상세

### 4.1 퀴즈 (quiz.test.js)

| 테스트 케이스 | 검증 내용 |
|--------------|----------|
| 데이터 무결성 | 8문제, 각 4지선다, 유효한 정답 인덱스 |
| 정답 판정 | 모든 문제 정답/오답 구분 정확성 |
| 점수 계산 | 전체 정답 100%, 전체 오답 0%, 부분 정답 |
| 퍼센트 계산 | 반올림 처리, 0으로 나누기 방어 |
| 결과 메시지 | 80↑ "훌륭합니다!", 60↑ "잘했어요!", 60↓ "다시 도전!" |
| 결과 이모지 | 점수대별 이모지 매핑 |

### 4.2 FAQ (faq.test.js)

| 테스트 케이스 | 검증 내용 |
|--------------|----------|
| 데이터 무결성 | 필수 필드 존재, 유효한 카테고리, 3개 카테고리 포함 |
| 카테고리 필터 | 진료실/접수실/보험청구 각각 정확히 필터링 |
| 키워드 검색 | 질문·답변 내 검색어 매칭, 대소문자 무관 |
| 복합 필터 | 카테고리 + 검색어 동시 적용 |
| 빈 검색 | 전체 반환 확인 |
| 미매칭 검색 | 빈 배열 반환 확인 |

### 4.3 시나리오 데이터 (scenario-data.test.js)

| 테스트 케이스 | 검증 내용 |
|--------------|----------|
| SCENARIO_ORDER | 6개, 중복 없음, 예상 순서 |
| 필드 완전성 | badge, title, desc, steps, tip 모두 존재 |
| 단계 수 | rx1:6, rx2:4, rx3:3, rec1:7, rec2:4, cl1:3 |
| 카테고리 그루핑 | 진료실 3개 먼저, 접수실 2개, 보험청구 1개 |

## 6. E2E 테스트 상세 (full-onboarding.spec.js)

| 시나리오 | 단계 |
|----------|------|

## 7. 테스트 실행 방법

```bash
# 의존성 설치
npm install

# 전체 단위+통합 테스트 (커버리지 포함)
npm test

# 단위 테스트만
npm run test:unit

# 통합 테스트만
npm run test:integration

# E2E 테스트 (Playwright 브라우저 설치 필요)
npx playwright install
npm run test:e2e

# 전체 테스트
npm run test:all
```

## 8. CI/CD 연계 계획

```yaml
# GitHub Actions 예시
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20' }
      - run: npm ci
      - run: npm test
      - run: npx playwright install --with-deps
      - run: npm run test:e2e
```
